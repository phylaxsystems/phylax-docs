## Overview

PCL (Phylax Credible Layer) CLI is a command-line tool for interacting with the Credible Layer. This guide covers all available commands and their usage.

For installation instructions, see the [Installation Guide](./credible-install.mdx).
For a getting started tutorial, see the [Quick Start Guide](./credible-quickstart.mdx).

## Global Options

```bash
pcl [OPTIONS] <COMMAND>

Options:
  -r, --root <ROOT_DIR>              The root directory for the project. Defaults to the current directory [env: PCL_ROOT_DIR=]
  -a, --assertions <ASSERTIONS_DIR>  The directory containing assertions 'src' and 'test' directories. Defaults to '/assertions' in the root directory [env: PCL_ASSERTIONS_DIR=]
  -h, --help                         Print help
  -V, --version                      Print version information
```

## Commands

### phorge

A minimal fork of Forge that includes a cheatcode for assertion execution.

```bash
pcl phorge [OPTIONS] <COMMAND>

Commands:
  test     Run tests
  script   Run scripts
  deploy   Deploy contracts
```

Example:
```bash
pcl phorge test
```

### build

Build the project and assertions.

```bash
pcl build [OPTIONS]
```

### store

Submit the Assertion bytecode and source code to be stored by the Assertion DA.

```bash
pcl store [OPTIONS] <ASSERTION>

Arguments:
  <ASSERTION>    Name of the assertion contract to submit

Options:
  --url <URL>    URL of the assertion-DA [env: PCL_DA_URL=]
```

Example:
```bash
# Store an assertion named "MyAssertion"
pcl store MyAssertion
```

### submit

Submit assertions to the Credible Layer dApp.

```bash
pcl submit [OPTIONS]

Options:
  -s, --project-name <NAME>     Name of the target project
  -a, --assertion-name <NAME>   Name of the assertion to submit
  --dapp-url <URL>              URL of the Credible Layer dApp [default: "https://credible-layer-dapp.pages.dev/api/v1"]
```

If not specified, the command will prompt interactively to select the project and assertion(s).

### auth

Authenticate with your Credible Layer dApp account.

```bash
pcl auth [OPTIONS] <COMMAND>

Options:
  --base-url <URL>    Base URL for authentication service [env: AUTH_BASE_URL=]

Commands:
  login     Login using your wallet
  logout    Remove stored credentials
  status    Check authentication status
```

## Project Structure

PCL expects the following directory structure:

```text
root-dir/
  assertions-dir/
    src/      # Assertion source files (.a.sol)
    test/     # Test files (.t.sol)
```

## Environment Variables

- `PCL_ROOT_DIR`: Set the root directory
- `PCL_ASSERTIONS_DIR`: Set the assertions directory
- `PCL_DA_URL`: Set the Assertion DA URL
- `AUTH_BASE_URL`: Override authentication service URL

## Configuration

PCL stores configuration in `~/.pcl/config.toml`. This includes:

- Authentication credentials (tokens and wallet address)
- Assertions prepared for submission
- User preferences

The configuration is automatically managed through CLI commands, particularly during authentication and assertion submission. Environment variables (like `PCL_ROOT_DIR`, `AUTH_BASE_URL`) will override settings from the config file.

## Support

For detailed help on any command:
```bash
pcl <command> --help
```

## Troubleshooting

Common issues and solutions:

### Authentication Errors
- **Error**: "Authentication required"
- **Solution**: Run `pcl auth login` to authenticate

### Build Failures
- **Error**: "Assertion compilation failed"
- **Solution**: Check assertion syntax and dependencies
